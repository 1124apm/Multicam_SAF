[ 챗봇 만들기 ]
1. 크롤링한 데이터를 통해, 팀별 특징을 정리한 JSON 생성
2. 수식 정의 (사용자 취향, 팀 키워드에 가중치를 두고 정확도 산출)
3. 가상의 사용자 데이터 (취향, 선호팀 키워드) 생성
4. 수식을 이용해 가상 데이터의 정답 생성
5. 가상 데이터(키워드)와 정답(추천팀)을 모델에 학습

[ 수식 ]
( alpha * 직관적 유사도 + beta * 잠재적 유사도 ) * 정체성 가중치
1. 직관적 유사도 (NLP 기반)
- 사용자 취향 키워드와 JSON의 팀별 특징 사이의 직접적 일치도
- TF-IDF 또는 SBERT를 이용한 텍스트 간 코사인 유사도
- Ex. "공격 축구" -> "공격" 태그를 가진 팀을 우선적으로 걸러내는 직관성 확보
2. 잠재적 유사도
- 직접적인 단어 일치보단, 그래프 구조상 유사한 환경이나 서사를 공유하는 정도
- Node2Vec으로 생성된 팀 벡터와 사용자 성향 벡터 간 코사인 유사도
- Ex. "열정적인 팬덤" -> 단어는 다르지만 유사한 응원 문화를 가진 팀 추천
3. 정체성 가중치
- 팀의 성적이나 인기도가 아닌, 감성적 데이터에 대한 가산점
- JSON 내의 특정 메타 데이터(Ex. 역사성, 드라마틱 지수, ...)를 0.9~1.1 사이의 가중치로 변환하여 곱
- 비슷한 점수의 두 팀 중, 사용자의 취향에 더 드라마틱하게 부합하는 팀을 최종 선정

[ 챗봇 - 전체 흐름 ]
1-1. 선호팀O: 사용자 취향 키워드 + 선호팀 키워드
1-2. 선호팀X: 사용자 취향 키워드
2. 'ML 모델'이 취향 키워드를 분석해 추천팀 산출
